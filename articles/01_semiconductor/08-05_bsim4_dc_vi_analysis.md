## はじめに

前回の記事では、**Paramus による BSIM4 モデル生成**を扱いました。  
モデルを作ったら、次にやるべきことは一つです。

> **モデルがどんな V–I 特性を出すのかを見る**

本記事では、BSIM4 モデルを用いた **DC 解析** により、

- \(V_g\)–\(I_d\) 特性  
- \(V_d\)–\(I_d\) 特性  

を確認し、MOSFET の基本動作を整理します。

---

## DC解析とは何か

DC解析とは、**時間変化を考えず、定常状態の電圧–電流関係（V–I）** を求める解析です。

MOSFET では主に次を見ます。

- ゲート電圧掃引：\(V_g\)–\(I_d\)  
- ドレイン電圧掃引：\(V_d\)–\(I_d\)

これらは、すべての回路設計・評価の出発点になります。

## 使用する環境（SemiDevKit）

本記事では、SemiDevKit に含まれる以下のモジュールを使用します。

- Python 実行環境

## Vg–Id解析（ゲート電圧掃引）

### 実行方法

```bash
cd bsim/bsim4_analyzer_dc/run
python run_vg.py
```

このスクリプトは、

1. SPICE ネットリスト生成  
2. ngspice 実行  
3. 結果の読み込み  
4. プロット表示  

を自動で行います。

---

### Vg–Id特性で分かること

横軸：ゲート電圧 \(V_g\)  
縦軸：ドレイン電流 \(I_d\)

このカーブから、次が読み取れます。

- しきい値電圧 \(V_t\)
- サブスレッショルド領域
- オン電流レベル
- ゲート制御の強さ

**TCAD で見た「チャネル形成」** が、  
ここで **V–I 特性として現れます**。

---

## Vd–Id解析（ドレイン電圧掃引）

### 実行方法

```bash
cd bsim/bsim4_analyzer_dc/run
python run_vd.py
```

---

### Vd–Id特性で分かること

横軸：ドレイン電圧 \(V_d\)  
縦軸：ドレイン電流 \(I_d\)

この特性からは、

- 線形領域（低 \(V_d\)）
- 飽和領域
- 出力抵抗 \(r_o\)

といった MOSFET の **動作領域** が確認できます。

---

## DC特性と物理の対応

DC解析で得られる V–I 特性は、完全にブラックボックスではありません。

- \(V_t\)：ドーピング・酸化膜厚  
- オン電流：移動度・チャネル長  
- 飽和特性：速度飽和・短チャネル効果  

これらはすべて、

> **TCAD → BSIM4 → SPICE**

という流れでつながっています。

---

## なぜDC解析が重要か

DC解析は、

- モデルが破綻していないか
- パラメータが妥当か
- 物理的におかしくないか

を最初に確認する **健全性チェック** でもあります。

AC / CV / 信頼性解析は、  
**DC特性がまともであることが前提** です。
