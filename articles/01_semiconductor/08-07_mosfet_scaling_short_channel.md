## はじめに

MOSFET は **小さくすると速くなる** と言われますが、  
同時に **壊れやすく・制御しにくく** なります。

本記事では、BSIM4 モデルを用いた **寸法スケール解析（L/W）** により、

- なぜ小さくすると特性が変わるのか
- 教科書的な直感と現実がどこでズレるのか

を **SPICE 解析結果から確認** します。

---

## 寸法スケーリングとは何か

寸法スケーリングとは、主に以下を変化させたときの影響を見る解析です。

- チャネル長：\(L\)
- デバイス幅：\(W\)

理想的には、

- \(I_d \propto W\)
- \(I_d \propto 1/L\)

と考えたくなりますが、**実デバイスではそう単純ではありません**。

---

## 使用する環境（SemiDevKit）

本記事では、以下のモジュールを使用します。

- BSIM4 Dimension Analyzer  
  https://samizo-aitl.github.io/SemiDevKit/bsim/bsim4_analyzer_dim/

前提：
- BSIM4 モデルカード
- ngspice
- Python 実行環境

---

## チャネル長 L のスケーリング

### 実行方法

```bash
cd bsim/bsim4_analyzer_dim/run
python run_dim_L.py
```

この解析では、チャネル長 \(L\) を段階的に変えながら  
DC 特性（V–I）を比較します。

---

### Lスケーリングで起きること

チャネル長を短くすると、次の現象が現れます。

- しきい値電圧の低下
- ドレイン誘起バリア低下（DIBL）
- 飽和特性の劣化
- 出力抵抗の低下

これらをまとめて **短チャネル効果** と呼びます。

---

## デバイス幅 W のスケーリング

### 実行方法

```bash
cd bsim/bsim4_analyzer_dim/run
python run_dim_W.py
```

---

### Wスケーリングの直感と現実

幅を広げると：

- 電流は増える
- ゲート制御性は大きく変わらない

ように見えますが、実際には：

- 寄生抵抗
- 寄生容量
- レイアウト依存効果

が無視できなくなります。

---

## BSIM4が果たす役割

BSIM4 は、

- 短チャネル効果
- 寸法依存パラメータ

をモデル内部で扱います。

そのため、SPICE 解析だけでも、

> **「小さくした結果、何が壊れたか」**

を定性的に確認できます。

---

## TCADとの関係

TCAD では、短チャネル化により：

- 電位分布が歪む
- 電界が強くなる

ことを直接観測できます。

BSIM4 の寸法スケール解析は、  
その結果を **端子特性として再現したもの** です。

---

## まとめ

- 寸法スケーリングは MOSFET 特性を大きく変える  
- L 縮小は短チャネル効果を引き起こす  
- W 拡大は単純な電流比例では終わらない  
- BSIM4 + SPICE で現実的な挙動を確認できる
